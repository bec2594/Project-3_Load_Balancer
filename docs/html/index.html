Project Overview

In this project I learned how to design and implement a model load balancer that manages incoming requests and dynamically adjusts web server allocation.

Load Balancer Behavior

The load balancer maintains a queue of incoming requests and distributes them to available web servers that process the requests. The system dynamically scales the number of servers depending on the queue size.

  • If the number of requests in the queue exceeds a threshold of 80 times the server count, the load balancer will add an additional server to help process the requests.
  • If the number of requests in the queue falls below 50 times the server count, the load balancer removes a server since it would otherwise remain idle.

Web Server Processing

Each web server accepts a request from the load balancer and processes it over the request's assigned processing time. Once the request is completed, the server returns the finished request and goes back to an idle state where it can accept another request.

Request Structure

A request is represented as an object containing:

  • An IP address
  • A job type
  • A processing time

Because each request contains an IP address, the load balancer can implement a firewall feature that blocks requests coming from specific IP address ranges.

The cycle time determines how long a request takes to complete, with a range between 10 and 100 cycles.
